# Política de solución de referencia

**Audiencia:** responsables del programa y equipo mentor
**Última revisión:** 2026-03-27

## Propósito

Este documento define el criterio recomendado para gestionar una posible solución de referencia del proyecto.

Su objetivo es permitir que el equipo formador disponga de una referencia funcional y evaluable sin exponer a las personas participantes una solución completa que reduzca el valor pedagógico de la práctica.

## Criterio general

No se recomienda publicar una rama con la solución completa dentro de este mismo repositorio.

Si se mantiene una solución de referencia, esta debe ubicarse en un repositorio privado separado o en un espacio interno accesible únicamente para mentores o personas responsables de la formación.

## Motivos

Este criterio busca:

- preservar la utilidad pedagógica del proyecto
- evitar la copia directa de la solución
- permitir al equipo formador disponer de una referencia cuando sea necesario
- facilitar la continuidad del programa sin exponer el resultado final a las personas participantes

## Recomendación operativa

Como enfoque preferente:

- mantener este repositorio como entorno de trabajo de las personas participantes
- mantener la solución de referencia en un repositorio privado independiente
- restringir el acceso a dicha solución al equipo mentor o a las personas responsables de la formación

## Alternativas complementarias

Si no se desea mantener una solución completa en código, pueden utilizarse también:

- checklist funcional de referencia
- capturas o recorrido guiado del resultado esperado
- notas de implementación para mentores
- ejemplos parciales de resolución por bloques

## Uso esperado

La solución de referencia no debe utilizarse como material de distribución para las personas participantes, sino como apoyo interno para:

- revisión de entregas
- contraste de decisiones técnicas
- resolución de dudas complejas
- mantenimiento del producto formativo a lo largo del tiempo
