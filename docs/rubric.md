# Rúbrica de evaluación

**Audiencia:** equipo mentor
**Última revisión:** 2026-03-27

## Propósito

Este documento define un criterio común para valorar las entregas de esta formación.

Su finalidad no es convertir la evaluación en un proceso rígido, sino aportar consistencia entre distintas personas evaluadoras y entre distintas ediciones del programa.

## Criterios de evaluación

### 1. Cumplimiento funcional

Se valora:

- que el alcance obligatorio esté implementado
- que los flujos principales funcionen de forma estable
- que la integración con backend sea correcta

### 2. Alineación con el contrato de API

Se valora:

- uso correcto de endpoints
- modelado adecuado de respuestas y payloads
- respeto a validaciones, paginación, filtros y roles

### 3. Calidad técnica

Se valora:

- organización del código
- legibilidad
- tipado
- separación razonable de responsabilidades
- ausencia de soluciones frágiles o difíciles de mantener

### 4. Experiencia de usuario

Se valora:

- claridad de navegación
- tratamiento de errores
- estados de carga
- feedback visual razonable
- consistencia básica de la interfaz

### 5. Comprensión y criterio

Se valora:

- capacidad para explicar lo implementado
- justificación de decisiones técnicas
- comprensión del uso de autenticación, roles y validaciones

### 6. Uso responsable de IA

Se valora:

- que la IA se haya utilizado como apoyo
- que exista comprensión real del resultado
- que no se haya delegado la solución completa de forma acrítica

## Niveles orientativos

### Entrega insuficiente

- no cubre el alcance obligatorio
- presenta fallos funcionales importantes
- muestra escasa comprensión del contrato o de la implementación

### Entrega básica

- cubre una parte sustancial del alcance obligatorio
- funciona con limitaciones o con falta de consistencia
- requiere apoyo adicional para considerarse cerrada

### Entrega sólida

- cubre el alcance obligatorio de forma correcta
- presenta integración coherente con backend
- mantiene una calidad técnica razonable
- permite defender las decisiones adoptadas

### Entrega destacable

- cubre el alcance obligatorio con solidez
- añade mejoras opcionales con criterio
- demuestra buena autonomía
- presenta un nivel superior de claridad, calidad o madurez técnica

## Recomendación de uso

La rúbrica no debe utilizarse como una lista mecánica de checks, sino como un marco de valoración global.

Una entrega no debería considerarse sólida únicamente porque “funciona”, si no existe comprensión suficiente o si el nivel de calidad es claramente inferior al esperado.
